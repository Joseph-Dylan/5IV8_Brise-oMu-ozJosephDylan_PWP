Todos los formularios en la web, manejan metodos para el envío de las diferentes peticiones que realiza el usuario. Cpnsiderando la arquitectura cliente-servidor, el manjejo de las peticiones se hace por medio del protocolo http y actualmente por https, cada petición debe de ser atendida en un tiempo de respuesta conforme al modelo tcp/IP. Los métodos principales de los formularios son:

    • Get:

      Este métodos se encarga de enviar la información el formulario por medio de la salida estandar (conodcida como url), del navegador, esto propicia que úedan exixisttir inyecciones de codigo realizando violaciones a ña seguridad del sistema

      POST:

      Este metodo se encargra de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema enviandola por el método estandar pero mediante un ocultamiento, esto signififca que no es visibvle para el ususario, lo que nos ofrece un mayor grado de sguridad en el manejo de la información


    • En resumen los metodos son:

    • GET: sirve para enviar y realizar opéraciones de lectura o consulta
      POST: sirve para enviar información de un recurso al servidor con el fin de registaralo
      Delete: sirve para eliminar un recursp especifico dentro del servidor, obteniendo ima respuesta de actuaización
      Put: sirve para almacenar la informacion de un recurso particular y concetarse al servicop para realizar la actualizacion
      Head: sirve para regresar el codigo de respuesta de cualquier cabecera http asociada a un recurso
